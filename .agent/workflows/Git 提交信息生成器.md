请帮我生成 Git 提交信息：

1. 使用 `git diff --cached` 或 `git diff HEAD` 查看当前暂存/工作区的变更
2. 深度分析变更内容：
   - 识别修改的文件和功能模块
   - 理解变更的业务意图和技术实现
   - 提取关键改动点
3. 生成简短凝练的中文提交信息（遵循 Git 最佳实践）：
   - 第一行：简短摘要（50字以内），清晰描述"做了什么"
   - 如有必要，第二行空行后补充详细说明（每行72字以内）
   - 使用动词开头（如：添加、修复、优化、重构、更新）

**可选增强**：如果我明确要求"对比上次提交"，请额外执行：
- 使用 `git log -1 --pretty=format:"%H%n%s%n%b"` 获取上次提交信息
- 对比本次变更与上次提交的关联性
- 生成更全面的提交描述

请直接输出生成的提交信息，格式清晰便于复制使用。
